# P2P Documentation: RubricsGuidelines.md

**What**: Expert rubric creation guidelines for R/M tasks
**Why**: Defines how to create rubrics with 15-20 atomic, objective, self-contained criteria
**Where**: .cortex/docs/RubricsGuidelines.md
**Learned**: 10 criteria requirements: Rubric-Specific (no overlap with scripts), Atomic (one idea each), Objective (PASS/FAIL), Correctly Framed (good=Yes/True), Positive phrasing, Accurate (match prompt), Scoped (only prompt requests), Self-Contained (no external refs needed), Unique (no overlap), Not Overfitting/Underfitting. 5 categories: Instruction Following, Code Correctness, Code Clarity, Efficiency, Risk Security. Weights only 1/3/5. No past tense, no questions, no "The model..." prefix.
