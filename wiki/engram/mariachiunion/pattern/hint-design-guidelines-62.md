# Hint Design Guidelines

Subtle Hints (preferred): sound like persona naturally framing request, reference data by product name not file path, point at desired outcome not technical fix. Specific Hints (fallback): explicitly name skills/tools/scripts, provide technical roadmap, use only after subtle hints failed across fresh reruns. NEVER: send hint as separate message, send follow-up hints, acknowledge hints in model response.
